# Project Rules & Specifications for liquidAmeba

このファイルは、`liquidAmeba` プロジェクトの設計思想、物理演算、音響合成（ASMR）、およびUI構築の黄金律を一元管理するための仕様書です。
Antigravity（AIエージェント）は、スレッドが新しくなった際も、**起動時にこのファイルを最優先で読み込み、開発状況を瞬時に同期します。**

---

## 1. UI & Design Rules (デザイン黄金律)

設定パネル（`#settings-panel`）を編集・拡張する際は、以下のレトロ1-bit Macテイストの厳格なUIルールを必ず遵守してください。

*   **フォントの統一**:
    *   すべてのテキスト（見出し、ラベル、ボタン）のフォントは **`Tiny5`** （Google Fonts）に統一します。
    *   他のピクセルフォント（Press Start 2Pなど）は使用しないでください。
*   **角丸の完全排除**:
    *   すべてのボタン、スライダー、パネル枠線において、角の丸め込みは一切禁止です（`border-radius: 0;` のみ、完全な直角）。
*   **補間アニメーションの完全排除**:
    *   UIのフェードイン/フェードアウト、変形などのCSS `transition` や `animation` はすべて禁止です。
    *   1-bitデジタルガジェットの小気味よい操作感を表現するため、状態変化はカチカチと「瞬時」に切り替わるようにします。
*   **目に優しいロールオーバー（反転色）**:
    *   ボタンがアクティブ（選択状態）の際の背景色は、就寝前の眩しさを防ぐために純白（#ffffff）ではなく、目に優しい半透明のライトグレー **`rgba(255, 255, 255, 0.7)`** を使用します。
*   **起動トリガーボタン (`#menu-trigger`)**:
    *   枠線なしのシンプルな正方形 glyph `□` です。通常時の不透明度は `0.8` とし、タップ時およびメニュー開時は同サイズの `■`（塗りつぶし）に瞬時に切り替わります（サイズ変更や白塗りは行わない）。
*   **クローズアクション**:
    *   クローズボタン（X）は廃止されています。設定ダイアログ内のインタラクティブ要素（スライダーやボタン）以外の余白背景を**タップ（click）**することで、スマートに閉じます。
    *   **重要**: タップとスクロール（ドラッグ）のバッティングを防ぐため、クローズイベントは `pointerdown` ではなく **`click`** にバインドし、スクロール時の意図しない誤作動クローズを防いでいます。

## 2. Physics & Rendering Mechanism (物理と描画システム)

アメーバの滑らかな1-bit描画と、有機的な追従物理は以下の仕組みで実装されています。

*   **1-bit SVGフィルター描画**:
    *   [index.html](file:///Users/kenjikishi/projects/liquidAmeba/prototype3/index.html) 内の `#smooth-one-bit` SVGフィルターでアンチエイリアスの効いた境界線を生成しています。
    *   `<feGaussianBlur id="base-blur">` の `stdDeviation` 属性（線の太さ）をJSから動的に操作することでアメーバの太さを制御します。
    *   砂嵐ノイズ（GRAIN）のブレンドは `<feComposite id="grain-blend">` の `k2` (ノイズ強度) と `k3` (アメーバ強度) のブレンド比で構成されます。
*   **有機的な物理イージング補間**:
    *   `LINE WIDTH`（線幅）と `GRAIN`（ノイズ）のゲージを変更した際、値は一瞬で切り替わらず、メインループ内で毎フレーム目標値（`target`）へ **`6%`** ずつじわじわと追従（イージング）します。これにより有機的で脳に優しい滑らかなアニメーションを実現しています。
*   **ATTRACTION（ポインター吸引）の完全連動**:
    *   `CONFIG.POINTER.PULL_FORCE` 定数（ATTRACTIONスライダーの設定値）は、以下の3つの物理挙動に完全連動しています。
        1. **引き寄せ力**: 静止した指にアメーバが引き寄せられる強さ。
        2. **追従係数（指の移動速度の継承度）**: 指を動かしたときにアメーバが指の動きにドラッグされて随伴する強さ（`js/physics.js`）。
        3. **離し際慣性**: 指を離した瞬間にアメーバに伝える移動慣性の強さ（`js/main.js`）。

## 3. ASMR Audio System (音響合成システム)

Web Audio API を使用し、ブラウザ上で完全動的シンセサイズされる心地よいASMR音響です。

*   **Drone (環境音)**:
    *   90Hzの超低域に低域カットオフを設定したおだやかなベースドローン音。指が画面に触れている時はダッキング（自動音量低下）が働きます。
*   **Water Flow (水流音)**:
    *   ホワイトノイズを急峻なバンドパスフィルター（BPF）に通し、指の移動速度（`Math.hypot(pointer.vx, pointer.vy)`）に同期して周波数を変動させ、水がうねる音を表現します。
*   **Bubble (水泡音)**:
    *   ポインターのタッチ時およびリリース時に、ランダムな周波数のサイン波と急峻な減衰エンベロープで合成された、ぷつぷつと弾ける気泡音を左右にパンニングさせて再生します。

## 4. Configuration & Maps (設定パラメータ)

すべての物理マップとデフォルト設定は [config.js](file:///Users/kenjikishi/projects/liquidAmeba/prototype3/js/config.js) の `CONFIG.MAPS` に定義されています。ユーザーが検証を重ねて決定した最終値は以下の通りです。

```javascript
MAPS: {
    SPEED: [0.0, 0.15, 0.45, 0.80],
    LINE_WIDTH: [4.0, 10.0, 30.0, 60.0],
    ATTRACTION: [0.0, 0.12, 0.35, 2.00],
    GRAIN: [0.0, 0.20, 0.40, 0.80]
}
```

## 5. Hidden Parameter Tuner (隠しチューナーの再活性化)

16個のパラメータをスライド微調整し、一括JSONエクスポート可能なチューニングコンソール（`.tuning-container`）は、製品版リリースに向けて非表示に設定されています。

*   **再表示方法**:
    *   [index.html](file:///Users/kenjikishi/projects/liquidAmeba/prototype3/index.html) 内の CSS 定義 `.tuning-container` にある **`display: none;`** の1行を削除する、またはブラウザのデベロッパーツールでこのスタイルを無効化するだけで、いつでもアコーディオン式チューナーが画面に出現し、再調整が可能です。

## 6. TUI & Observer Console (観察モニターとトラッキング枠)

赤ちゃんのエコー検査や有機液体のスキャンモニターを思わせる、目に優しく落ち着いたアスキーアート/テキスト情報のUIシステムです。

*   **３つの角への分散配置による没入感**:
    *   画面の周囲に情報を分散させることで、アメーバが動く中央エリアを全く邪魔せず、測定器のディスプレイ枠のような佇まいを表現しています。
    *   **左上 (SYS)**: UPTIME（稼働秒数）、BLOBS（個体数）、AVG TMP（平均温度）、SYS BPM（全体脈拍）、FLOW（水流速度）。
    *   **左下 (FOCUS)**: スキャン中の個別個体の生体情報（SIZE, TEMP, PULSE）。
    *   **右下 (LOG)**: タッチ（`POINTER ACTIVE` / `RELEASED`）や結合（`FUSION`）、分裂（`MITOSIS`）のタイムスタンプ付きイベント履歴。
*   **ガタつき（チラつき）防止の行数固定**:
    *   フォーカス個体が一時的に未選択（`FOCUS: NONE`）のときも、`SIZE: --.- MM` / `TEMP: --.- *C` などのプレースホルダー行を出力し、パネル全体を常に4行に固定することで、テキスト行数が変化して高さがガタつく現象を完全に防いでいます。
*   **SVGフィルターに影響されないHTMLオーバーレイ**:
    *   メインアメーバを描画する Canvas には強力なSVGフィルター（`#smooth-one-bit` によるぼかし＋二値化）がかかっています。
    *   この影響を受けずに、Tiny5フォントの極小文字（PC: `10px`, モバイル: `8px`）や、トラッキング枠の点線をシャープに描画するため、スキャナーやテキストは Canvas 外のHTMLオーバーレイ（`#observer-console`, `#tracking-frame`）としてレンダリングされています。
*   **トラッキング解析枠のハイブリッド追従物理**:
    *   フォーカス個体に合わせて表示される四隅ブラケット（`┌ ┐`）と中心ドット・IDラベルです。
    *   **位置（x, y）の即時移動**: 個体の切り替え時や通常移動時、枠がスライドする余計なアニメーションを排除し、一瞬で「パッと」その場所へジャンプ移動します（1-bitテイストの堅牢なTUI感）。
    *   **サイズ（r / 広がり）のイージング追従**: 枠の大きさ（`track.r`）に対しては `8%` のイージングを維持し、個体が融合した際や個体差によるサイズ変更時には、枠の大きさが「すーっ」と吸い付くように滑らかに拡縮します（機械的かつ有機的な心地よさ）。
*   **タッチ連動ターゲットロックオン**:
    *   指を離している間（通常時）は、5秒ごとに巡回ランダムスキャンが自動で行われます。
    *   画面をタッチしている間（ポインターアクティブ時）は自動巡回を一時停止し、**「ポインターに最も近い（＝一番反応している）アメーバ」をリアルタイムでロックオン**します。指を動かして別のアメーバへアプローチすると、枠がパッパッとターゲットを切り替えて追従します。指を離すと、タイマーが再開し自動巡回に戻ります。
*   **設定（SETTINGS）画面のTUIデザイン統合**:
    *   音量スライダーの左右に括弧 `[` と `]` を配置し、トラックを点線（`radial-gradient`）、つまみを文字カーソル風の白い矩形（`█`）に変更。右端にリアルタイムに同期する数値ラベルを設置しました。
    *   物理調整ボタン（`gauge-btn`）は、無骨な白枠（ボーダー）をすべて排除し、ホバー時の輝度アップとタップ時の白黒反転アクションだけにすることで、TUIスライダーとの静かな融和を図っています。
